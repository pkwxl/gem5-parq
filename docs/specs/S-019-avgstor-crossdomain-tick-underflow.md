# S-019 — `AvgStor` 跨域 Tick 下溢：修法与偏差实测

- **状态**：初版（Architect，2026-07-22）——研究点待 Researcher 深化
- **分支**：`s019-avgstor-crossdomain-tick-underflow`（尚未创建）
- **承接**：S-018 §6.2（根因定位，未修）
- **治理决策**：[0002 — `AvgStor` 跨域倒退 Tick 的处理方式](../decisions/0002-avgstor-backward-tick-clamp.md)
- **优先级**：P0（[ROADMAP](../roadmap/ROADMAP.md)，先于出路 3）

## 1. 背景与目标

### 1.1 为什么是 P0

出路 3（摊薄跨域同步固定税）是当前唯一直击 0.459x 的方向，而它的成败**只能靠
测量判定**。S-018 §6.2 定位到的下溢发生在 `AvgStor`——`statistics::Average` /
`AverageVector` 的共享后端，全仓库 23 处声明、9 个文件——意味着**并行臂的任何
时间加权统计当前都不可信**。不先修，出路 3 的全部实验数据带毒。

**本 spec 不产出任何加速比数字。** 这一点在 ROADMAP 和 INDEX 里都写明了，此处
再记一次：它是一次 correctness 工作，不得被计入出路 3 的进展。

### 1.2 要回答的问题

1. 钳位修法能否消除 `k * 2^64 / D` 假值，且**不改变串行臂的任何数字**？
2. 钳位引入的偏差有多大——具体说，是否小到让受影响统计仍然可用？
3. 除 `MessageBuffer` 外的另外 8 个文件是否同样受影响？

问题 2 是本 spec 的重心。决策 0002 §3.3 已经指出一个不利的可能：偏差的理论
上界（每量子倒退更新次数 × 一条消息）**可能与被测量本身同量级**——串行
`m_buf_msgs` 真值只有 0.05–0.11 条消息。分析到此为止，再往下是经验问题。

## 2. 根因（承接 S-018 §6.2，此处只作自足复述）

`Tick` 是 `uint64_t`；`AvgStor` 三处方法各做一次 `curTick() - last` 的减法；
`last` 由最后碰过该统计对象的那个域盖戳。放宽跨域时序下，落后的域更新共享统计
量时 `curTick() < last`，相减下溢成 ~2^64，乘以当时占用数 k，得到精确等于
`k * 2^64 / D` 的假值（已观测 6 个，k ∈ {1,2,3,5}，分母 D 完全正确——这一点
**排除**了「分母取错域的 `curTick()`」这个曾经的首选假说）。

偏斜有界：屏障保证所有域落在同一量子窗口内，故 `last - curTick() < Q`
（当前工作点 Q = 6660 tick）。

代码落点（Architect 读代码新增，S-018 §6.2 只点到 `set()`）：

| 文件:行 | 内容 | S-018 是否记录 |
|---|---|---|
| `src/base/stats/storage.hh:156` | `set()` 的减法 | 是 |
| `src/base/stats/storage.hh:201` | `prepare()` 的同一减法（dump 前调用） | **否** |
| `src/base/stats/storage.hh:186` | `result()` 的 `assert(last == curTick())` | **否** |

`result()` 那条断言是**尚未被观测到、但代码上必然存在**的第二症状：领先域盖戳
后，dump 线程读到更小的 `curTick()`，断言在带断言的构建下直接触发。当前跑法
未撞上，不代表不存在。

## 3. 研究点拆解

以下四点相互独立，Researcher 可逐个深化，编号在后续会话中沿用。

### R1 — 钳位修法的实现与串行惰性验证

按决策 0002 §4 实现三处改动。**串行惰性是可证明的**（0002 §3.2 性质 1：串行
恒有 `t_i >= m_{i-1}`，钳位分支永不进入），R1 的任务是把这条证明变成实证——
串行臂 `stats.txt` 改动前后逐字节相同。

### R2 — 偏差的实测量化（本 spec 的重心）

决策 0002 §4 强制要求的插桩：钳位分支必须计数并记录 `Σδ`、`max δ`、以及
`Δv` 的符号分布（区分倒退入队与倒退出队——0002 §3.2 性质 3 指出，只有当两者
不对称时才产生系统性偏差，方向为**高估占用**）。

R2 要产出的是一个能和 §1.2 问题 2 对上的数字：受影响统计的**估计偏差占其自身
值的百分比**。判据见 §5。

### R3 — 假值消除的确认

并行臂不再出现 `k * 2^64 / D` 量级的值。注意 S-018 §6.1 已记录：**哪些 link
受影响、有几条，每次跑都在变**——所以「这次没看到」不等于「修好了」。R3 需要
一个比单次观察更强的判据，设计留给 Researcher。

### R4 — 其余 8 个文件的暴露面审计（只审计，不修）

`cpu/o3/lsq_unit.hh`、`mem/mem_ctrl.hh`、`mem/cache/tags/base.hh`、
`cpu/simple/exec_context.hh`、`mem/ruby/structures/{TBEStorage,MN_TBEStorage}.hh`、
`mem/ruby/system/GPUCoalescer.hh`、`dev/arm/ufs_device.hh`。判定各处的
`Average` 是否跨域共享。修法在 `AvgStor` 层已经全覆盖，R4 的价值是**知道哪些
既有统计数字曾经可能被污染过**——这会影响如何解读此前 S-NNN 的并行臂数据。

## 4. 设计思路

修法本身见决策 0002 §4，不在此复述。三点补充：

- **A1 而非 A2**：`current = val` 必须无条件执行。跳过它会让统计对象的占用数
  与缓冲区真实占用永久失配，比下溢更糟（0002 §3.1）。
- **插桩不是可选项**。没有 R2 的数字，0002 §3.2 的误差表达式只是一个无法核验
  的说法，本 spec 也就无法结题。
- **不要在本 spec 里顺手做出路 3 的任何事**。两者的耦合点只有一个：R2 的结论
  决定出路 3 的测量能不能信。

## 5. 验收标准（预先写死）

按 `CLAUDE.md` 工作风格，判据在跑之前定死，跑完不得修改。

| # | 判据 | 不满足时 |
|---|---|---|
| V1 | 串行臂 `stats.txt`（排除 `host*`）改动前后**逐字节相同** | 修法不惰性 → 实现有误，返工 |
| V2 | 并行臂不再出现 `k * 2^64 / D` 量级假值，且判据强于单次观察（R3） | 返工 |
| V3 | 受影响统计的估计偏差 **≤ 其自身值的 10%** | **A 不予接受**，回到决策 0002 改选 D（把这些统计标注为并行模式下不可比） |
| V4 | 热路径开销：并行臂 `hostSeconds` 变化在噪声内 | 记录实测值，不阻塞结题 |

**V3 的 10% 是拍的。** Researcher 有权在实验计划里论证换一个阈值，但必须在
跑之前定死并写进 spec（决策 0002 §5）。

**三臂口径**：本 spec 不涉及加速比，V1/V2 只需 Current Serial 与 Current
Parallel 两臂（Baseline 不含本 fork 的跨域机制，不存在倒退更新，比较无意义）。
这是 `CLAUDE.md`「三臂比较」允许的例外，代价是**本 spec 的数据不能用来说明任何
关于加速比的事**——V4 的 `hostSeconds` 只作开销参考，不是 speedup 测量。

## 6. 未决问题

- V3 的阈值是否合理，由 Researcher 在实验计划里定。
- R2 的插桩以什么形式落地（`statistics` 自身的计数器？`warn_once`？独立
  CSV，沿用 S-012 的 critpath 通路？）——影响是否需要额外的构建开关。
- 若 V3 不满足而改选 D，「标注为不可比」的具体机制是什么（stats 输出加标记？
  只在文档里约定？）——届时需要一份新的决策记录，不在本 spec 范围内。
- 本 spec 未涉及：`AvgStor` 之外是否还有其他 `uint64_t` Tick 相减的跨域读。
  同族问题（S-009…S-016）的历史表明这类点是逐个冒出来的，值得一次独立审计，
  但那是新编号，不塞进本 spec。
