# 0004 — 保留核：单点定义、开跑前实测空闲、重核任务通则

- 状态：已采纳（2026-07-22）
- 相关：[0003 角色门三处裁定](./0003-role-gate-scons-merge-and-repo-config.md)、
  `CLAUDE.md`「Measurement methodology」与角色红线清单、`util/roles/reserved-cores`、
  `.claude/skills/check-cores/`

## 1. 背景

三个问题同时暴露，共用一份修法。

**(a) 核号是散落的字面量。** `54-55`/`92-111` 以字面量形式出现在 `role-gate.py`、
`CLAUDE.md`（4 处）、4 份角色 `PROTOCOL.md`（7 处）、`AGENTS.md`、`QWEN.md`。换机器、
改隔离核配置、或临时让出一组核，都要人肉全仓库搜索替换；漏一处就是一条与其余文档
矛盾的纪律，而这类矛盾恰恰只会在实验跑到一半时才被发现。

**(b) 「保留」不等于「空闲」。** 我们在容器里，`ps`/`top` 只看得见本容器的进程。
宿主机或另一个容器把任务绑到我们的隔离核上时，这里**完全看不见**，而它照样会污染
一整轮 A/B 计时。原 Experimenter 协议写的「确认保留核当前空闲（`ps`/`top` 核查）」
在这个环境下是一条**无效**的纪律——它检查的对象根本不包含真正的威胁。

**(c) 只有 `scons` 被管住。** 门里唯一的重核任务规则是「`scons -j` 必须绑核」，而且
它的正则 `-j\s*\d+|-j\d+` 只匹配数字：**`scons -j$(nproc)` 直接放行**，而 `nproc`
在本容器返回 112，正好吃满含保留核的整机。S-010 §12 记录过这次事故，门却一直没堵上。
`ninja`（默认并发）、`make -j`、`pytest -n`、`xargs -P`、`tests/main.py -j` 同样不管。

### 一次现场验证

写完采样脚本后第一次跑（2026-07-22，采样 2s）：串行臂两个核与并行臂的
`92-99` 共 10 个核**实测 100% 忙**，而同一时刻本容器 `ps -eo pid,psr,pcpu` 里
最高占用是 6.4% 的 `claude` 进程本身，没有任何进程在这些核上。**(b) 描述的盲区不是
假设，是当场复现的事实**：不知情地在此时开跑，两条臂都会拿到被污染的数字，而按旧
协议做的 `ps` 检查会显示一切正常。

## 2. 缺口 (a) —— 单点定义

**备选方案 A（否决）**：从 `/proc/cmdline` 的 `isolcpus=` 运行时推导。否决理由：它给
不出**分工**——哪组给串行臂、哪组给并行臂、哪组留给构建，是这个项目的约定而不是内核
的信息；而且容器的 cgroup cpuset 未必等于宿主的 isolcpus（S-009 §25.2 记录过取不到
的情形），推导结果会随环境静默漂移。

**决策**：新增 `util/roles/reserved-cores`，`KEY=VALUE` 格式（shell 可 `source`，
Python 三行可解析），含 `SERIAL_ARM_CPUS`、`PARALLEL_ARM_CPUS`、`BUILD_CPUS`、
`IDLE_SAMPLE_SECONDS`、`IDLE_MAX_BUSY_PCT`。三个下游读它：`role-gate.py`、
`use-role`（切角色时回显当前值）、`check-cores` skill。散文（`CLAUDE.md`、四份
`PROTOCOL.md`、`AGENTS.md`、`QWEN.md`）改为**指名这个文件、不再复述数字**。

**失败安全方向**：文件缺失或缺键时，门对任何带绑核参数或属于重核任务的命令返回
`ask`，**不回落到内置默认**——一个静默失效的保留核门比没有门更危险，因为它给出的是
虚假的安全感。`use-role` 在同样情形下打印警告。

`docs/specs/**` 里的核号**不动**：那是历史实验记录，记的是当时实际用的核，随本文件
变动去改写它们等于篡改实验记录。同理 0001/0003 两份决策记录里的字面量也保留原样。

## 3. 缺口 (b) —— 开跑前实测空闲

**关键事实**：`/proc/stat` **没有被 PID namespace 隔离**。从容器内读到的是宿主机全部
112 个逻辑核的累计时间——正好覆盖 `ps` 看不见的那部分。这是整个方案成立的支点。

**判据**：连续采样 `IDLE_SAMPLE_SECONDS`（默认 10）秒，单核忙碌率
`(Δtotal − Δidle − Δiowait) / Δtotal` 低于 `IDLE_MAX_BUSY_PCT`（默认 0.1%）才算空闲。
`steal` 计入忙碌——被 hypervisor 抢走的时间同样拿不到 CPU，对计时的污染没有区别。
本机保留核同时设了 `nohz_full` 与 `rcu_nocbs`（见 `/proc/cmdline`），真空闲时连周期
时钟中断都没有，实测就是 `0.000%`，所以 0.1% 是现实阈值而非理想值；换到没有
`nohz_full` 的机器上需要放宽，改配置文件即可。

**备选方案 B（否决）**：把它做成硬门——`role-gate.py` 记录 skill 的检查结果到状态
文件，绑核到保留核的 gem5 命令若无近期通过记录则 `deny`。**用户拍板否决**，理由成立：
门无法验证检查的**新鲜度**（10 秒前空闲不等于开跑那刻空闲），状态文件本身又成了新的
可绕过点，得到的是虚假的安全感而不是安全。

**决策**：`check-cores` skill 只**报告**，判定与裁决权交给人。落地为纪律：
Experimenter 的 Phase A 第 3 条从「`ps`/`top` 核查」改为「跑 `check-cores`，把逐核
实测值贴进 Checkpoint 1」，并新增一条硬红线——判定为「忙」时**不得自行改用别的核**
开跑，如实报告等用户裁决。

技术上必须连同结论一起说出口的两条限制（skill 自己每次也会打印）：占用率是**证据**
不是**归属**（能证明有人在用，证明不了是谁，也证明不了没有人已经绑定但此刻恰好空
转）；且它**不保鲜**。

## 4. 缺口 (c) —— 重核任务通则

**备选方案 C（否决）**：凡未经 `taskset`/`numactl` 的命令一律 `ask`。否决理由：把
`grep`/`ls`/`git` 这类日常调用全变成确认提示，与门自己「护栏不是沙箱，假想敌是漂移和
手滑」的定位冲突，且高频误报会训练出「一律确认」的条件反射，反而削弱真红线。

**决策**：显式列出会吃满整机的工具，**逐工具绑定**它的并行旗标（这一点很要紧：
`xargs -n` 是批大小、`pytest -n` 才是并发度，用同一个宽正则会误杀）：

| 工具 | 触发条件 |
|---|---|
| `scons`、`make`、`tests/main.py` | 出现 `-j`（含裸 `-j`、`-j80`、`-j $(nproc)`） |
| `ninja` | 无条件——它默认就并发 |
| `xargs` | `-P N`（N≠1） |
| `pytest` | `-n N` 或 `-n auto` |

命中且未绑核 → `deny`，提示里直接给出当前 `BUILD_CPUS` 的值。`-j` 的正则同时从
`-j\d+` 放宽到 `-j\s*\S+`，堵上 `-j$(nproc)` 这个真实事故的入口。

## 5. 后果与已知代价

- **`BUILD_CPUS` 与两条臂的互斥没有任何代码强制**，是人工维护的三元组：改了臂却忘了
  改 `BUILD_CPUS`，门会一边禁止你碰保留核、一边建议你绑到一个含保留核的范围。
  `test-role-gate.py` 末尾新增三条断言（两条臂不重叠、`BUILD_CPUS` 不含保留核、三键
  非空）作为兜底——**改配置后必须跑一次测试**，这是本决策唯一依赖人执行的环节。
- 采样 10 秒是**每次实验前的固定开销**，且期间不能做别的判断。相对动辄数小时的三臂
  跑可以忽略，但对「快速试一下」的场景是真实摩擦；`--seconds` 可临时压缩，代价是样本
  变弱，压缩这件事本身应当在 Checkpoint 1 里说明。
- 重核任务清单是**白名单式**的：清单外的新并行工具（例如将来引入 `bazel`、`cargo -j`）
  回落到放行，且是**静默**放行。这与 0003 §5 记的根级配置文件白名单是同一类已知
  缺陷——列表法的代价。
- `check-cores` 是纪律不是机制，门不会因为没跑它而拦人。这是 §3 备选 B 被否的直接
  后果，明确接受。

## 6. 何时应重新审视

- 若出现「实验跑完才发现保留核当时被别人占着」的实例，说明只靠纪律不够，应重新考虑
  备选 B（带新鲜度窗口的硬门）。
- 若换到没有 `nohz_full`/`rcu_nocbs` 的机器，0.1% 阈值会被时钟中断顶穿，届时应重新
  标定阈值而不是简单调大——先测一台确认空闲的机器的本底噪声。
- 若清单外的并行工具造成第二次「吃满整机」事故，说明白名单模式不成立，应重新考虑
  备选 C（默认 `ask` + 常用只读命令白名单）。
