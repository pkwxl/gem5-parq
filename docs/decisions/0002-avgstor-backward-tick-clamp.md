# 0002 — `AvgStor` 跨域倒退 Tick 的处理方式

- **状态**：已接受（待 S-019 的实测门槛核验；见 §6「接受是有条件的」）
- **日期**：2026-07-22
- **相关**：S-018 §6.2（根因定位）· S-019（本决策的落地）· ROADMAP P0
- **同族**：S-009 / S-014 / S-015 / S-016 —— 「读另一个域盖戳的状态」这一类

## 1. 问题

`AvgStor`（`src/base/stats/storage.hh`）是 `statistics::Average` /
`AverageVector` 的后端，做时间加权平均。三个方法各有一次同样的减法：

```cpp
// set()  :156        // prepare():201
total += current * (curTick() - last);
last = curTick();
// result():186
assert(last == curTick());
```

`Tick` 是 `uint64_t`。本 fork 放宽了跨域时序，共享同一个统计对象的两个域对
`curTick()` 不取得一致；当落后的域更新一个由领先域盖过戳的统计量时
`curTick() < last`，`curTick() - last` 下溢成 ~2^64，再乘以当时的占用数 k，
产出精确等于 `k * 2^64 / D` 的假值（S-018 §6.2 观测到 6 个，k ∈ {1,2,3,5}，
分母 D 完全正确）。

范围不止 Ruby：`statistics::Average` / `AverageVector` 全仓库 23 处声明、
9 个文件，含 `cpu/o3/lsq_unit.hh`、`mem/mem_ctrl.hh`、`mem/cache/tags/base.hh`。
`prepare()` 的同一处下溢和 `result()` 的断言此前未被记录——只修 `set()` 会漏。

**偏斜是有界的**：屏障保证所有域落在同一个量子窗口内，所以
`last - curTick() < Q`（当前工作点 Q = 6660 tick）。这个界是下面全部分析的基础。

## 2. 备选方案

- **A. `AvgStor` 内钳位** —— `curTick() <= last` 时跳过累加与 `last` 推进，但
  **仍然执行 `current = val`**。
- **B. 在 `MessageBuffer` 调用点修** —— 只治已观测到的那一个症状。
- **C. 域一致时钟** —— 把时间戳锚到量子边界（`simQuantumStart + k*simQuantum`，
  `eventq.hh:79`，本 fork 已有 grid-anchored snap 成例）。
- **D. 承认并排除** —— 不修，把受影响的时间加权统计标注为「并行模式下不可比」，
  从三臂对照的比较集里剔除。

B 直接否决：`AvgStor` 是共享上游代码，另外 8 个文件同样暴露；出路 3 一旦提高
并发度就会陆续冒出同类假值，逐点修等于把同一个 bug 修 23 遍。剩下 A / C / D。

## 3. A 的偏差分析（本决策的核心）

### 3.1 形式化

设按**执行顺序**发生的更新为 $(t_i, v_i)$：$t_i$ 是发起更新的那个域的
`curTick()`，$v_i$ 是更新后的占用数。$t_i$ 跨域不单调。

A 的行为可以精确刻画。令 $m_i = \max(t_1,\dots,t_i)$（即 `last` 在第 $i$ 步后
的值，单调不减）：

- $t_i > m_{i-1}$：`total += ` $v_{i-1}\,(t_i - m_{i-1})$，$m_i = t_i$
- $t_i \le m_{i-1}$：不累加，$m_i = m_{i-1}$
- 两种情况都执行 `current = ` $v_i$

**语义等价刻画**：A 等于把每次更新的时间戳**前推到运行最大值**——一次真实
发生在 $t_i$ 的占用变化，被记成发生在 $m_{i-1} \ge t_i$。占用值本身一个不丢，
丢的只是它的**发生时刻**。

这也解释了为什么必须保留 `current = val`（方案 A 的 A1 变体）。若连
`current` 一起跳过（A2），统计对象的占用数会与缓冲区真实占用**永久失配**，
之后每一次前进更新都会把这个错误占用数加权到一整段时间上——那比下溢还糟。
**A2 明确否决。**

### 3.2 误差表达式

设 $\delta_i = m_{i-1} - t_i \ge 0$ 为第 $i$ 次更新的前推量，
$\Delta v_i = v_i - v_{i-1}$。在 $[t_i, m_{i-1}]$ 这段上 A 用了旧值
$v_{i-1}$ 而非 $v_i$，于是积分误差

$$\mathrm{err} = -\sum_i \Delta v_i\,\delta_i,\qquad 0 \le \delta_i < Q$$

由此得三条性质：

**性质 1（串行惰性，可证明）**。串行模式下只有一个 `curTick()`，恒有
$t_i \ge m_{i-1}$，即 $\delta_i \equiv 0$，$\mathrm{err} = 0$，且钳位分支永不
进入。**修法在串行臂上逐字节惰性**——此前每一份 S-NNN 记录的串行统计数字
一个不变。这是 A 相对 C 最重要的优势：它不要求重新解释任何既有结论。

**性质 2（有界性）**。$|\mathrm{err}| \le \sum_i |\Delta v_i|\,Q$。对
`MessageBuffer` 每次入队/出队 $|\Delta v_i| = 1$，故每个量子内的误差
$\le n_{\text{lag}} \cdot Q$，其中 $n_{\text{lag}}$ 是该量子内**倒退**更新的
次数。分母每量子增长 $Q$，所以**平均占用数的相对误差上界约等于每量子的倒退
更新次数**（单位：条消息）。

**性质 3（抵消 vs 系统性偏差）**。$\Delta v_i$ 在缓冲区上取 $\pm 1$。若倒退
更新中入队与出队等概率，$\mathrm{err}$ 在期望上**相消为零**，只剩方差。但若
存在系统性关联——典型情形是**消费侧的域系统性落后**，于是倒退更新绝大多数
是出队（$\Delta v_i = -1$）——误差变成
$\mathrm{err} = +\sum \delta_i > 0$，即**占用数被系统性高估**（出队被记晚了）。
这是 A 唯一的真实偏差，方向可预测：**朝高估占用**。

### 3.3 这个偏差有多大？——不利的一面

性质 2 的界必须和被测量本身的量级比。S-018 §6 记录的串行 `m_buf_msgs` 真值
在 **0.05–0.11 条消息**这个量级。而按性质 2，每量子哪怕只有 1 次平均前推量
$Q/2$ 的倒退出队，高估量就是 **0.5 条消息**——比被测量本身还大一个量级。

**所以 A 不是「显然安全」的。** 它的可接受性完全取决于一个此刻未知的经验量：
共享统计对象上倒退更新的**频次**与 $\Delta v$ 的**符号分布**。分析到此为止，
再往下推就是猜。

### 3.4 与 C、D 的对比

**C（量子边界时钟）在同一个尺度上更差**，不是更好。C 下同一量子内的多次更新
$\delta = 0$，累加恒为 0——占用变化在量子内**全部丢失**，`total` 退化成每量子
一次的零阶保持采样。即：A 的误差只在**倒退**更新上出现（性质 1 保证顺序更新
精确），C 的误差在**每一次**量子内更新上都出现。若 A 因 §3.3 的量级问题不可
接受，C 必然也不可接受——两者的误差同为 $O(Q)$ 量级，而 C 的触发面严格更大。
C 还额外丢掉性质 1：它会改变串行臂的数字，此前每个 S-NNN 的统计都要重新解释。

**D（承认并排除）是 §3.3 成立时的真正兜底**。如果实测证明倒退更新频繁到让
偏差与被测量同量级，那么诚实的结论是：**在放宽跨域时序的前提下，跨域共享的
时间加权统计本身就没有良定义的值**，任何钳位/量化都只是把一个显式的错误换成
一个隐式的错误。那时应当做的是把这些统计标注为并行模式下不可比、从三臂对照
的比较集中剔除，而不是发明一个看起来正常的数字。

## 4. 决策

**采纳 A（A1 变体）**，覆盖三个点：

1. `set()` —— `Tick now = curTick(); if (now > last) { total += current * (now - last); last = now; }`，`current = val` 无条件执行。
2. `prepare()` —— 同一守卫（此前未被记录的第二处下溢）。
3. `result()` —— `assert(last == curTick())` 放宽为 `assert(last <= curTick())`
   或在并行模式下取消。这是一个**尚未被观测到、但代码上必然存在**的第二症状：
   领先域盖戳后，dump 线程读到更小的 `curTick()`，断言在带断言的构建下会直接炸。

**同时强制要求插桩**（这是本决策不可分割的一半）：钳位分支必须计数并记录
$\sum \delta_i$、$\max \delta_i$、以及 $\Delta v$ 的符号分布。没有这组数字，
§3.2 的误差表达式就只是一个无法核验的说法。

## 5. 接受是有条件的

本决策**不**断言 A 产出的数字是可信的。它断言的是：A 是四个方案里唯一能
**同时**做到「串行惰性」「消除 2^64 假值」「把偏差变成可测量而非可假设」的。

S-019 必须预注册一条判据并实测：若在均衡检查点的真实窗口上，受影响统计的
**估计偏差超过其自身值的 10%**，则 A 不予接受，回到本决策改选 **D**——
把这些统计标注为并行模式下不可比。10% 这个阈值是拍的，S-019 的 Researcher
有权在实验计划里论证换一个，但必须在跑之前定死。

## 6. 后果与代价

- 串行臂：无任何可观测变化（性质 1）。既有 S-NNN 结论不受影响。
- 并行臂：`k * 2^64 / D` 量级假值消失；受影响统计带一个方向已知（偏高估）、
  幅度待实测的偏差。
- 热路径：每次 `AvgStor::set()` 多一次可预测分支。`Average` 类统计共 23 处，
  其中 `MessageBuffer::m_buf_msgs` 在 Ruby 热路径上——开销需实测，但与
  `UncontendedMutex` 的既有取舍同一量级。
- **本决策不产出任何加速比数字**，也不得被计入出路 3 的进展。

## 7. 何时重新审视

- S-019 的实测偏差超过 §5 的阈值 → 改选 D。
- 出路 3 改变了量子长度 Q → 性质 2 的界随 Q 线性变化，需重新核验。
- 若将来引入真正的域一致时钟（不是 C 那种量子量化，而是每域独立累加、
  dump 时合并），本决策整体作废——那是比 A 更根本的解法，代价是重做
  `AvgStor` 的存储布局。
